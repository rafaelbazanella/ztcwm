## v3.1 Requirements — Outcomes

### Member Visibility

- [x] **MEMBER-01**: Network detail screen displays each member's installed ZeroTier client version alongside the online/offline status indicator
  - **Outcome:** Validated — closed by Phase 18 (commit chain e78696a..d8e8e6e), verified 2026-05-11
  - **Implementation:** Inline version sub-line (`<span class="version">· vX.Y.Z</span>`) appended to existing `<zt-badge>` status indicator. Status column widened 90px → 180px to fit `v999.999.999`. Wired through new `MemberWithPeer extends Member { version?: string }` view type and `memberService.listMembersWithPeers(networkId)` service method.

- [x] **MEMBER-02**: When the client version is unavailable (member offline, controller has not reported a version yet), the version cell shows a neutral placeholder (e.g. `—`) — no `undefined`, no `null`, no perpetual spinner
  - **Outcome:** Validated — closed by Phase 18, verified 2026-05-11
  - **Implementation:** Three-state matrix locked: offline rows hide the sub-line entirely (D-05); online + unknown shows `· —` em-dash (D-06); D-07 detection regex `/^0\.0\.0(\.|$)/` strips uninstalled-client placeholders at both the service layer (member-service.ts:64-66) and defensively at the render site (network-detail.ts:539). No `<zt-loading>` or spinner on the version path. Tests B-UI-1, B-UI-2, B-UI-3 lock all three states.

### Internationalization

- [x] **I18N-01**: Every visible UI string in the SPA (components, pages, modals, toasts, error messages, button labels, table headers, empty states) is in English — no remaining Portuguese strings
  - **Outcome:** Validated — closed by Phase 19, verified 2026-05-12
  - **Implementation:** Two-pass `src/scripts/i18n-audit.sh` (accent class + 43-token alternation under `LC_ALL=C.UTF-8`) exits 0 against current `src/` tree. Opt-in npm row `audit:i18n` enables re-runnable verification. 69-row Admin/Operator/Viewer/Unauthenticated walkthrough matrix in `19-AUDIT.md` (D-04 row-by-row discipline overridden via `/gsd-next --force`; reviewer Rafael Bazanella self-attested with concrete dates).

- [x] **I18N-02**: User-visible backend response messages (validation errors, audit-log lines surfaced in the UI) are in English
  - **Outcome:** Validated — closed by Phase 19, verified 2026-05-12
  - **Implementation:** Same audit script covers `src/server/routes/*.ts` and `src/server/auth/*.ts` (recursive grep with D-02 excludes). Walkthrough rows 24, 25, 43, 44, 51, 52 exercise 4xx code paths (out-of-route IP, duplicate IP, duplicate username, wrong-password reset).

### Users Page

- [x] **USERS-01**: Users-page action buttons for **edit**, **reset password**, and **delete** display Lucide icons (via `lucide-static`), restoring the visual standard broken in the v3.0 `.btn-*` migration
  - **Outcome:** Validated — closed by Phase 20 plan 20-01, verified 2026-05-15
  - **Implementation:** Added `sharedStyles` to `<zt-data-table>` static styles (position 2 of the array, mirroring `<zt-modal>`'s pattern). The `.btn-icon svg { width: 16px; height: 16px }` rule from `src/styles/shared.ts:103-106` now crosses the shadow-DOM boundary into per-cell renders. D-03 nested-shadow regression test (`users.test.ts:193-216`) iterates all 3 buttons asserting both width AND height per Lucide SVG.

### Layout

- [x] **LAYOUT-01**: `<zt-navbar>` renders outside the page outlet (`<div id="outlet">`) so it stays mounted across navigation — page content alone re-renders on route change
  - **Outcome:** Validated — closed by Phase 20 (plans 20-02, 20-03, 20-04, 20-05), verified 2026-05-15; UAT-2 + UAT-3 confirmed 2026-05-19
  - **Implementation:** `<zt-navbar>` mounted once inside `<zt-app>.render()` at `src/app.ts:143` (non-login branch only). Route metadata (`title`, `subtitle`) on all 10 authenticated routes flows via `vaadin-router-location-changed` event detail. `<zt-app>.setTheme(target, options?)` is single writer of `<zt-app>.theme` `@state`; navbar's `currentTheme` is parent-bound `@property`. 13 per-page `<zt-navbar>` invocations + 10 dead navbar.js imports deleted. CR-01 BLOCKER closed by construction (Option B / IN-04). Two regression tests in `app.test.ts:261-317` (CR-01 guard) + `:319-366` (System UX guard).

- [x] **LAYOUT-02**: Navbar aligns vertically with the brand logo container (`<div class="brand">`) at the top of the viewport — no gap, no offset
  - **Outcome:** Validated — closed by Phase 20 plan 20-03 + three UAT-driven follow-up fixes (commits 8f622f1, f1aa201, f979081), verified 2026-05-15 + UAT-1 confirmed 2026-05-19/20
  - **Implementation:** Closing fix combined three commits:
    - 8f622f1: stacked-content height correction (initial UAT gap closure)
    - f1aa201: `.nav-title-stack` height-locked (28px, overflow: hidden) — prevents Inter line-metric drift through `line-height: 1`
    - f979081: `<zt-navbar>` `:host` padding reduced from `1rem 1.25rem` → `0.83rem 1.25rem` (closing fix, determined empirically by reviewer in DevTools)
  - **Note:** The byte-for-byte geometry mirror approach proved fragile; future header-alignment work should consider a shared `--header-band-padding` token (deferred — no second consumer yet justifies the abstraction).

## Out of Scope (Reaffirmed — Outcome)

| Feature | Reason | Outcome |
|---------|--------|---------|
| Multi-language support / runtime locale switching (i18n framework like i18next) | English is the single chosen language for a self-hosted admin tool | Reaffirmed — no i18n framework introduced; strings remain inline literals |
| Replacing `lucide-static` with a different icon library | Existing icon system stays; standardization is to adopt it everywhere | Reaffirmed — `lucide-static` is the established standard |
| Sidebar layout / structure changes | This milestone only fixes navbar position; sidebar's existing collapse + grouping is correct | Reaffirmed — sidebar untouched |
| Real-time client-version updates (push/poll/SSE) | Manual refresh remains the established pattern | Reaffirmed — `loadData` re-fetches every navigation (D-15) |
| New CI guardrail test for English-only strings | Could be added later; not requested for v3.1 | Reaffirmed — audit:i18n is opt-in only (D-06) |
