# Phase 18 ‚Äî UI Design Contract

> Visual and interaction contract for the Member ZT Client Version inline metadata. Generated by gsd-ui-researcher from CONTEXT.md decisions D-01..D-08; verified by gsd-ui-checker.

## Typography

The version token is **muted secondary metadata** that visually defers to the primary status badge. It is rendered in the existing monospace family because it is a version literal (matches the existing `physicalAddress` mono treatment two columns over), at one step below the body size.

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| Body (existing) | 14px / `0.875rem` | 400 | 1.4 (inherits) | `--font-size-base` |
| Label (existing) | 13px / `0.8125rem` | 500 | 1.4 | `--font-size-sm` |
| Heading (existing ‚Äî page title) | 24px / `1.5rem` | 700 | 1.2 | `--font-size-2xl` |
| **Version token (this phase)** | **12px / `0.75rem`** | **400** | **1.4** | `--font-size-xs` + `var(--font-mono)` family |
| Separator `¬∑` | 12px / `0.75rem` | 400 | 1.4 | inherits from version token wrapper |

**Rationale:**
- Monospace for `v1.10.6` and `‚Äî` matches the existing convention (`physicalAddress`, `nodeId`) for machine-emitted strings.
- `--font-size-xs` (12px) sits one step below the row's body size (`--font-size-sm` 13px in `td`), reinforcing its secondary-metadata role per D-03.
- Weight 400 (regular) ‚Äî the row's default; bumping to 500 would compete with the badge.

**Total declared sizes for this phase (1 new):** `--font-size-xs` for the version token + separator. All other text in the cell uses pre-existing typography.

**Total declared weights for this phase (0 new):** weight 400 only.

## Copywriting Contract

This phase is **inline metadata, not a workflow** ‚Äî it has no buttons, no destructive actions, no empty-state heading. The copy below is exhaustive for what Phase 18 introduces.

| Element | Copy | Source decision |
|---------|------|-----------------|
| Online member, version available | `v1.10.6` (with `v` prefix + raw `peer.version`) | D-02 |
| Online member, version unavailable | `‚Äî` (single em-dash, U+2014) | D-06, D-07 |
| Offline member | (sub-line absent ‚Äî render only the existing `Offline` badge, no separator, no placeholder) | D-05 |
| Inline separator between status and version | `¬∑` (U+00B7 MIDDLE DOT) ‚Äî only present when a version token is rendered (i.e. online rows only per D-05) | D-03 |
| Loading / pre-fetch state | (treat as version unavailable ‚Äî render `‚Äî` for online rows, nothing for offline rows; no spinner) | D-08 |
| Peer-fetch graceful degrade (network/5xx error fetching `/peer`) | Same as version unavailable: online rows render `‚Äî`, offline rows render nothing | D-14 |

**Copy specifics (locked):**

- Version literal is **never** prefixed with parentheses, never wrapped, never abbreviated. Patch segments are preserved (D-02: no `v1.10` truncation).
- Em-dash is the U+2014 character (`‚Äî`), not two hyphens (`--`) and not the U+2013 en-dash (`‚Äì`).
- The version is **plain text** ‚Äî not a link, not a tooltip trigger, not interactive (no hover state, no click handler).

**No copy in this phase for:**
- Primary CTA ‚Äî phase has no buttons
- Empty state ‚Äî phase does not change existing members-empty rendering
- Error toast ‚Äî D-14 leaves toast emission to plan-phase discretion; if plan-phase chooses to emit one, it must be a transient advisory ("Could not fetch peer versions") and must NOT block rendering. The version-cell copy contract above stands regardless of toast presence.
- Destructive confirmation ‚Äî phase has no destructive actions

## States Matrix

The version token has exactly **3 visual states**. State 4 (offline) is "absent", not "rendered".

| State | Trigger | Visual | A11y |
|-------|---------|--------|------|
| 1. Online + version known | `online === true` AND `peer.version` is non-empty AND does not match `/^0\.0\.0(\.|$)/` | `[Online badge] ¬∑ v1.10.6` (separator + mono muted token) | Version is regular text inside the row; row already has a label-bearing `Status` column header |
| 2. Online + version unknown | `online === true` AND (`peer` missing OR `peer.version` empty OR `peer.version` matches `/^0\.0\.0(\.|$)/`) | `[Online badge] ¬∑ ‚Äî` (em-dash in place of version) | Em-dash is conventional "no data"; no extra labelling required |
| 3. Offline | `online === false` | `[Offline badge]` (no separator, no placeholder, no whitespace bump) | Existing Offline badge already conveys state |

**No state for:**
- Loading ‚Äî D-08 collapses loading into State 2 (online) or State 3 (offline) until data arrives. There is no spinner, no skeleton, no "loading‚Ä¶" string anywhere on the version sub-line.
- Errored ‚Äî D-14 collapses peer-fetch failure into State 2 / State 3. Errors are non-blocking; the version cell degrades gracefully.

## Layout & Placement

**Anchor (D-01):** version is rendered on the **same line** as the existing online/offline status badge in the existing `Status` column of the members `<zt-data-table>`. No new column. No stacked sub-line.

**Container shape:** the `Status` column's `render` callback returns an inline-flex wrapper that holds the existing `<zt-badge>` first and (conditionally) the `¬∑ version` group second.

**DOM contract for the cell (online + known version, illustrative ‚Äî exact HTML is plan-phase territory):**

```html
<span class="status-cell">
  <zt-badge variant="success">Online</zt-badge>
  <span class="version">¬∑&nbsp;v1.10.6</span>
</span>
```

**DOM contract for offline (D-05):** only `<zt-badge variant="error">Offline</zt-badge>` ‚Äî the `<span class="version">` is omitted entirely (Lit's `nothing` sentinel, not an empty string).

**Column width:** the existing `Status` column is `width: '90px'` (line 519 of `network-detail.ts`). Phase 18 widens this column to accommodate `Online ¬∑ v1.10.6`. The new width is plan-phase territory; the contract requires only that the cell **does not wrap** under typical version strings (‚â§ 12 characters: `v` + `1.10.6` ‚âà 8 char body; allow up to `v999.999.999`). Estimate: **~170‚Äì190px** column width.

**Wrapping rule:** the inline-flex wrapper uses `white-space: nowrap` on the `.version` span so a long version (`v1.10.6-rc1`) does not break mid-token. If the column is too narrow, the *cell* may overflow horizontally (handled by `.table-wrapper`'s `overflow-x: auto`); the version itself never wraps.

## Accessibility

The version is decorative metadata adjacent to a labelled status badge. v3.1 does not include an accessibility milestone (per `PROJECT.md` Out of Scope: keyboard nav / ARIA deferred). However, this phase MUST NOT regress existing a11y posture:

- **Reading order:** `Status` column header ‚Üí `Online`/`Offline` text ‚Üí separator ‚Üí version (or em-dash). The separator `¬∑` reads as "middle dot" in most screen readers, which is acceptable for muted metadata.
- **No new ARIA labels are required** for the version token. The existing `Status` column header (rendered by `<zt-data-table>`) is the labelling anchor for the entire cell.
- **Color is not the only signal:** version is rendered as text, not as a color-coded chip. Muted color is a hierarchy hint, not an information channel ‚Äî passing `--color-text-muted` already meets WCAG AA contrast on both themes (verified by `theme-contrast.test.ts`).
- **No focusable elements added** ‚Äî the version is not interactive.

**Optional (Claude's Discretion, plan-phase):** an `aria-label="ZeroTier client version 1.10.6"` on the `.version` span is a low-cost a11y enhancement. Not required by the contract.
